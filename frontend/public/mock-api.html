¿Qué es una Mock API?

Una mock API (o API falsa) es una interfaz de programación simulada que imita el comportamiento de una API real pero sin backend. Devuelve respuestas predefinidas con los status codes, headers y JSON que tú configures.

Es la herramienta perfecta para desarrolladores que necesitan testear aplicaciones, prototipar interfaces o desarrollar agentes de IA sin depender de servicios externos costosos o inestables.

Mock API vs API Dummy

Existen varios términos relacionados:

  • Mock API: Simula el comportamiento completo de una API real con rutas, métodos y respuestas.
  • Dummy API: Generalmente devuelve datos simples y estáticos sin lógica. Es más básica.
  • Fake API: Término amplio que incluye mocks y dummies. Se usa de forma intercambiable.
  • Stub API: Devuelve respuestas predefinidas para tests unitarios. Más simple que un mock.

¿Cómo crear una Mock API con MockAgent?

  • 1. Define el endpoint: Elige una ruta como /api/orders y el método HTTP
  • 2. Configura la respuesta: Escribe el JSON con los datos que quieras devolver
  • 3. Elige el status code: 200, 201, 400, 401, 404, 500... cualquiera
  • 4. Publica: Obtén una URL pública al instante
  • 5. Integra: Usa la URL en tu frontend, tests o agente de IA

Casos de uso de APIs de Prueba

  • Desarrollo Frontend: Tu UI necesita datos antes de que el backend esté listo
  • Testing E2E: Simula APIs de terceros en tests automatizados sin depender de su disponibilidad
  • Prototipado: Valida ideas con stakeholders mostrando una app funcional con datos reales
  • Agentes de IA: Entrena agentes que consumen APIs sin depender de servicios externos
  • Formación: Enseña a juniors cómo consumir APIs sin necesidad de un backend complejo
