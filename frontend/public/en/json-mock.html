What is a JSON Mock?

A JSON mock is a simulated response in JSON format that mimics the data a real API would return. It is the fundamental component of any mock server or test API.

Instead of waiting for the backend to be ready, you generate a JSON with the exact structure you need and your frontend or AI agent consumes it as if it were real.

JSON Mock Example

Imagine you need to simulate a users API. With MockAgent.AI you create this JSON:

{ "users": [ { "id": 1, "name": "John Doe", "email": "john@example.com", "role": "admin", "active": true }, { "id": 2, "name": "Jane Smith", "email": "jane@example.com", "role": "user", "active": false } ], "total": 2, "page": 1 }

And you get a public URL: https://tfg-mockagent-production.up.railway.app/mock/api/users

Types of JSON Mocks You Can Create

Simple JSON Mock

A basic object with few fields. Ideal for quick tests and simple endpoints.

JSON Mock with Arrays

Lists of JSON objects. Perfect for simulating collections of users, products, orders, etc.

Nested JSON Mock

Objects within objects. Useful for complex APIs with relationships (user → company → departments).

JSON Mock with Errors

Simulate validation errors with status 400 and error JSON: {"errors": [{"field": "email", "message": "Invalid format"}]}
